# Test Plan - Sauce Web Application Demo

## 1. Objective
The objective of this test plan is to carify the core functionality, usability and reliability of the Sauce Demo web application. The focus is on ensuring that key user 
journeys work as expected and that defects are identidied and documented clearly.

-----

## 2. Application Under Test
- Name: Sauce Application Demo
- URL: https://www.saucedemo.com
- Type: Web-based e-commerce demo application

-----

## 3. Scope

### IN Scope
- Login Functionality
- Product listing page
- Add/Remove products from cart
- Cart Overview
- Checkout Process
- Logout Functionality

### Out of Scope
- Performance Testing
- Security Testing
- Accessability Testing
- Backend/Database Testing
- Payment Processing (Mock Functionality)

-----

## 5. Test Environment
- Browser: Google Chrome (Current Version: 143.0.7499.169/170)
- Operating System: Windows 11
- Device: Laptop
- Network: Truespeed home Broadband

-----

## 6. Test Data
The following test credentials will be used:
- Username: standard_user
- Password: secret_sauce

-----

## 7. Entry and Exit Criteria

### Entry Criteria
- Application is accessable via the URL
- Test Credentials are available
- Test cases are written and reviewed

### Exit Criteria
- All planned Test cases executed
- All Hight and Medium severity defects logged
- Test Execution completed and documented

-----

## 8. Risks and Assumptions
- As a ublic demon application functionality may change without notice.
- Limited test data available
- No access to backend logs or database

-----

## 9. Deliverable

- Test Plan Document
- Test Cases
- Bug Report
- Exploratory Testing Notes
- Test Execution evidence tracked via Trello

-----

## 10. Tools Used
- Trello (For test tracking and defect management)
- GitHub (Test Documentation and Version Control)
- Google Chrome (Browser Testing)

-----

## 11. Approval
This Test Plan is created for portfolio and learning and to demonstate structured QA testing.
